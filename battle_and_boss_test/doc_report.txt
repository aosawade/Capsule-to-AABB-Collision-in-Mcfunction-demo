# Battle_and_Boss_Test 数据包分析报告

## 概述
这是一个为 Minecraft 1.21.11 设计的战斗与 BOSS 测试数据包，实现了胶囊体与 AABB 的精确碰撞检测系统，并集成了玩家动作系统（跳跃下半身无敌和闪避无敌）。数据包与 Animated Java 动画系统集成，支持动态碰撞检测。

## 数据包结构分析

### 命名空间架构
1. **`collision_check`** - 碰撞检测核心系统
   - `function/loop.mcfunction` - 主循环检测入口
   - `function/collision_sys_init.mcfunction` - 系统初始化
   - `function/colli_success_func.mcfunction` - 碰撞成功回调
   
2. **`globle`** - 全局工具函数
   - 坐标变换工具（相对坐标 ↔ 展示实体坐标）
   - 位置/矢量转换函数

3. **`minecraft`** - 游戏集成
   - `tags/function/load.json` - 加载时函数
   - `tags/function/tick.json` - 每 tick 执行函数

4. **`player_action`** - 玩家动作系统
   - 跳跃下半身无敌帧
   - 闪避无敌帧与冷却系统

## 碰撞检测系统详细分析

### 核心算法：相对坐标胶囊体 ↔ AABB 检测
1. **检测流程**：
   ```
   loop → run_as_player → load_aabb → run_as_capsule → load_capsule → cap2aabb_check → ray2box_check
   ```

2. **技术实现**：
   - **坐标系转换**：使用相对坐标系统，以玩家为原点建立坐标系
   - **胶囊体表示**：由起点、方向矢量、长度和半径定义
   - **AABB 扩展**：通过半径扩展 AABB，将胶囊体检测转换为射线检测
   - **射线-AABB 相交算法**：使用 Slab Method 计算相交区间

3. **关键算法文件**：
   - `ray2box_check.mcfunction` - 实现了精确的射线-AABB 相交检测
   - `cap2aabb_check.mcfunction` - 胶囊体到 AABB 的转换逻辑

### 玩家状态系统
- **状态码**：0=正常，1=全身无敌，2=下半身无敌
- **AABB 尺寸配置**：
  - 正常状态：宽 0.6 格，高 1.8 格
  - 跳跃状态：下半身高度减少

## 玩家动作系统分析

### 跳跃下半身无敌
1. **检测机制**：使用 `minecraft.custom:minecraft.jump` 计分板
2. **无敌时长**：默认 10 tick
3. **状态切换**：检测到跳跃 → 状态设为 2 → 计时器递减 → 归零恢复

### 闪避无敌系统
1. **冷却机制**：10 tick 冷却，分段恢复
2. **魔咒载体**：使用皮革马铠作为效果载体
3. **状态管理**：通过 `dodge_effect_time` 和 `dodge_cd` 控制

## 全局工具函数分析

### 坐标变换系统
1. `transform_pos.mcfunction` - 位置矩阵变换
2. `transform_vec.mcfunction` - 矢量矩阵变换
3. `pos_rel2display.mcfunction` - 相对坐标到展示实体坐标转换

### 关键技术亮点
- 使用展示实体的 `transformation` 矩阵进行坐标变换
- 支持 Animated Java 动画系统的动态碰撞检测
